# Безопасность и приватность - Локальное хранение псевдонимов

## Обзор изменений

Система была обновлена для полной приватности и защиты от атак:

### 1. **Локальное хранение псевдонимов** 🔒
- **ДО**: Псевдонимы (mappings) сохранялись на сервере в папке `/static/pseudonym_maps`
- **ПОСЛЕ**: Псевдонимы хранятся **только в браузере пользователя** (localStorage)
  - Сервер больше не имеет доступ к личным данным пользователя
  - Каждый пользователь управляет своими данными независимо

### 2. **Автоматическое удаление** 🗑️
- Все псевдонимы **автоматически удаляются через 30 дней** (настраивается)
- Пользователь видит оставшееся время до удаления каждого mapping'а
- При каждой загрузке приложения проверяются и удаляются истекшие данные

### 3. **Защита от атак** 🛡️

#### Rate Limiting (ограничение частоты запросов)
- `/anonymize`: **30 запросов в минуту** на IP-адрес
- `/deanonymize`: **30 запросов в минуту** на IP-адрес
- `/generate`: **20 запросов в минуту** на IP-адрес
- `/csv-headers`: **30 запросов в минуту** на IP-адрес
- `/validate-json`: **30 запросов в минуту** на IP-адрес

Защита от:
- DDoS атак
- Brute-force атак
- Перегрузки сервера

#### Валидация размеров файлов
- **Максимальный размер файла**: 50 МБ
- **Максимально строк в CSV**: 1,000,000
- Автоматическая проверка и отклонение слишком больших файлов

#### Валидация входных данных
- Все параметры проверяются перед обработкой
- Поддерживаемые кодировки: UTF-8, UTF-8-SIG, CP1251
- Строгая проверка типов данных

### 4. **Управление диском** 💾
- **Ограничение сервера**: 13 ГБ
- Сервер **не хранит пользовательские данные**, только временно обрабатывает их
- Все файлы после обработки удаляются из памяти
- Значительное снижение требований к дисковому пространству

## Архитектурные изменения

### Бэкенд (main.py)
```python
# ДО
- save_pseudonym_mapping() → сохранял на disk
- load_pseudonym_mapping() → загружал с disk
- mapping_id возвращался в заголовке X-Pseudonym-Mapping-Id

# ПОСЛЕ
- Mapping отправляется прямо клиенту в JSON ответе
- Клиент сохраняет все данные локально
- Бэкенд не хранит никаких pseudonym data
```

### API Изменения

#### POST /anonymize (Pseudonymization)

**Ответ - ДО:**
```
CSV file + Header: X-Pseudonym-Mapping-Id
```

**Ответ - ПОСЛЕ:**
```json
{
  "csv": "email,name\npseudo_xxx,John\n...",
  "mapping": {
    "columns": {
      "email": {
        "pseudo_xxx": "user@example.com",
        "pseudo_yyy": "admin@example.com"
      },
      "name": {
        "pseudo_aaa": "John Doe",
        "pseudo_bbb": "Jane Smith"
      }
    }
  }
}
```

#### POST /deanonymize

**Запрос - ДО:**
```
CSV file + Form: mapping_id
```

**Запрос - ПОСЛЕ:**
```
CSV file + Form: mapping (JSON string)
```

### Фронтенд (Anonymize.jsx)

**Хранение - ДО:**
```javascript
// Хранил только ID
{
  id: "e7f8c3a2b1d4e5f6",
  createdAt: "2024-01-01T10:00:00Z",
  sourceFile: "data.csv"
}
```

**Хранение - ПОСЛЕ:**
```javascript
// Хранит полный mapping + метаданные
{
  id: "m_1234567890_abc123",
  mapping: { /* полные данные для восстановления */ },
  createdAt: "2024-01-01T10:00:00Z",
  expiresAt: "2024-01-31T10:00:00Z",  // Автоудаление через 30 дней
  sourceFile: "data.csv"
}
```

## Инструкции по запуску

### Перед обновлением
```bash

```

### Установка зависимостей
```bash
# Обновлены requirements.txt
```

### Запуск с Docker
```bash

docker-compose up --build
```

### Запуск локально
```bash
# Backend
cd app
python -m uvicorn main:app --reload

# Frontend (в отдельном терминале)
cd frontend
npm install
npm run dev
```

## FAQ

### Где сохраняются мои псевдонимы?
Они хранятся **только** в `localStorage` вашего браузера (обычно в папке профиля браузера). Сервер не имеет доступа к этим данным.

### Что происходит с моими данными, когда я закрываю браузер?
Данные остаются в браузере. При следующем открытии браузера вы сможете продолжить работу. Через 30 дней они автоматически удаляются.

### Могу ли я передать mapping другому пользователю?
Технически можно вручную экспортировать JSON из localStorage, но это не рекомендуется по соображениям безопасности. Каждый пользователь должен создавать свои mapping'и.

### Какой максимальный размер файла?
**50 МБ** на один файл, **1 миллион строк** максимум.

### Что если я превышу rate limit?
API вернет ошибку 429 (Too Many Requests). Попробуйте снова через минуту.

### Как контролировать дисковое пространство?
Сервер не использует дисковое пространство для хранения данных пользователя. Все данные обрабатываются в памяти и удаляются после обработки.

### Могу ли я изменить время удаления (30 дней)?
Да! В `frontend/src/pages/Anonymize.jsx` найдите:
```javascript
const MAPPING_EXPIRY_DAYS = 30 // Измените это значение
```
